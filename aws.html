AWS Services

Our AWS architecture for the Gen AI system was comprehensive. We used Amazon Lex for the conversational interface, allowing IT staff to interact with the system via voice or text. Incoming queries were processed by Lex and sent to a Lambda function, which orchestrated the workflow. For heavy lifting, we used SageMaker to host our fine-tuned Megatron-LM model, leveraging its automatic scaling. Historical tickets and model checkpoints were stored in S3 for durability. We used Amazon RDS (PostgreSQL) to store structured data like incident categories, user feedback, and application logs. EC2 instances handled web services and real-time analytics.

AWS Lex

Developed conversational AI chatbots using AWS Lex to automate customer service and support processes, integrating with backend systems for data retrieval and updates.

AWS SageMaker

Trained and deployed machine learning models, including the anomaly detection and recommendation engines, using SageMaker's managed infrastructure and integrated tools for data processing, model training, and hosting.

AWS EC2

Provisioned and managed compute instances (EC2) to run resource-intensive workloads, such as distributed model training, data processing pipelines, and containerized application deployments.

AWS S3

Utilized S3 (Simple Storage Service) as a secure and scalable data lake to store and retrieve large volumes of structured and unstructured data required for model training and inference.

AWS RDS

Employed RDS (Relational Database Service) to set up and manage relational databases like MySQL and PostgreSQL for storing and querying student data, course materials, and other application data.

AWS Lambda

Implemented serverless functions using AWS Lambda to build event-driven architectures, such as real-time data processing pipelines, API gateways, and integrating various AWS services.

AWS API Gateway

Created and managed RESTful APIs using API Gateway, acting as an entry point for client applications to interact with backend services like machine learning models and databases.

AWS CloudWatch

Monitored and analyzed application performance, resource utilization, and system health using CloudWatch's log aggregation, metrics collection, and alerting capabilities.

AWS IAM

Implemented robust access controls and permissions management using AWS IAM (Identity and Access Management) to ensure secure access to resources and services across different environments.
