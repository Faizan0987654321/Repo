real-time data ingestion from MySQL to Power BI

We used AWS Database Migration Service (DMS) to replicate our MySQL database to an Amazon Aurora instance in near real-time. This handled the load without impacting the production database. Next, we set up AWS Data Pipeline to transform and aggregate data into a star schema optimized for analytics. The processed data was then loaded into Amazon Redshift, which Power BI could query directly using its optimized connector. We also used AWS CloudWatch to monitor replication lag and query performance.

insights

"In Power BI, we focused on actionable insights. For individual students, we had 'learning path' flowcharts showing topic progressions and highlighting bottlenecks. Heatmaps displayed time distribution across subjects, revealing neglected areas. For instructors, we had classroom-level dashboards with engagement metrics, topic mastery bell curves, and predictive charts showing atrisk students. We used lots of conditional formatting, like red-yellow-green scales for performance. The most loved feature was the 'concept cloud,' where size represented difficulty and color showed mastery level.

Student Insights Dashboard

Overall Performance Metrics

  • Average Grades: Displays the student's average grades across all subjects.
  • Subject-Specific Grades: Detailed view of grades for each subject.

Attendance Records

  • Total Attendance: Percentage of classes attended.
  • Subject-Specific Attendance: Attendance rates for each subject.

Performance Trends

  • Grade Trends: Visualization of the student's grade trends over time, showing improvements or declines.
  • Attendance Trends: Visualization of attendance patterns over time.

Strengths and Weaknesses

  • Subject Proficiency: Highlights the subjects where the student excels.
  • Areas for Improvement: Identifies subjects and topics where the student is struggling.

Engagement Metrics

  • Study Time: Amount of time spent on the platform or specific subjects.
  • Material Usage: Details on the usage of recommended study materials (e.g., videos watched, articles read).

Recommendation Effectiveness

  • Resource Utilization: Tracks which recommended resources the student has used.
  • Impact on Performance: Measures how the usage of recommended materials has impacted the student's grades and understanding.

Assessment Scores

  • Quiz and Test Scores: Detailed scores from quizzes and tests taken on the platform.
  • Assignment Performance: Scores and feedback on assignments submitted.

Behavioral Insights

  • Learning Patterns: Insights into the student's learning habits and behaviors (e.g., peak study times, preferred learning methods).
  • Interaction Data: Frequency and type of interactions with the platform (e.g., discussion participation, question asking).

Progress Towards Goals

  • Goal Setting and Achievement: Tracks the student's progress towards set academic goals.
  • Milestones Reached: Significant milestones or achievements in the learning journey.

Feedback and Notes

  • Instructor Feedback: Comments and feedback from instructors on the student's performance.
  • Student Self-Assessment: Any self-assessment data or reflections provided by the student.

Comparison with Peers

  • Performance Comparison: Comparison of the student's performance with the class average or specific peer groups.
  • Engagement Comparison: How the student's engagement metrics compare to peers.
